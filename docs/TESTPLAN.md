# COBOL Account System Test Plan

This test plan covers the current business logic and runtime implementation of the COBOL account application (`MainProgram`, `Operations`, and `DataProgram`).

| Test Case ID | Test Case Description | Pre-conditions | Test Steps | Expected Result | Actual Result | Status (Pass/Fail) | Comments |
|---|---|---|---|---|---|---|---|

Prompt requests choice `(1-4)`. | TBD | TBD | Validates entry point behavior in `MainProgram`. |
| TC-002 | View initial balance | Application is running; no prior credit/debit in current run | 1. Enter `1` at menu prompt. | System displays `Current balance: 001000.00` (format may include leading zeros). | TBD | TBD | Validates initial in-memory balance is 1000.00. |

## Notes for Stakeholder Validation

- This plan intentionally includes both business-rule tests and implementation-observation tests so stakeholders can decide target behavior for the Node.js modernization.
- For migration readiness, mark each test as one of:<br>`Preserve as-is`, `Change behavior`, or `Clarify with business`.
